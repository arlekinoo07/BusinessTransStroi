# Priority Scoring

В проекте приоритетность сделок считается по шести индексам в диапазоне `0..5`:

- `Need Score` — реальность и конкретность потребности
- `Time Score` — близость окна мобилизации
- `Spec Score` — четкость ТЗ
- `Access Score` — близость к ЛПР
- `Money Score` — готовность к коммерческой обработке и оплате
- `Fit Score` — рентабельность для своего парка

## Формула

1. Считается среднее по шести индексам:

```text
average = (need + time + spec + access + money + fit) / 6
```

2. Среднее переводится в проценты относительно шкалы `0..5`:

```text
maturity_percent = round((average / 5) * 100)
```

## Категории зрелости

- `Горячая`: `> 80%`
- `В процессе`: `40%..80%`
- `Холодная`: `< 40%`

## Где это хранится в Bitrix24

Приоритетность записывается не в комментарии, а в отдельные поля сделки:

- `UF_CRM_NEED_SCORE`
- `UF_CRM_TIME_SCORE`
- `UF_CRM_SPEC_SCORE`
- `UF_CRM_ACCESS_SCORE`
- `UF_CRM_MONEY_SCORE`
- `UF_CRM_FIT_SCORE`
- `UF_CRM_MATURITY_PERCENT`
- `UF_CRM_MATURITY_STAGE`

Комментарии теперь содержат только краткую служебную сводку, а сами значения score живут в полях сделки, чтобы их можно было выводить отдельными колонками в списке Bitrix24.
